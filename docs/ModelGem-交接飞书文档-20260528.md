## 1. 项目定位

Model Gem 是一个面向 TikTok 内部跨团队协作的模型效果评测与选型平台 Demo。它把图像模型、视频模型、评测题库、横向对比、需求匹配和接入指引整合到一个网站里，目标是让设计、产品、算法、开发和业务同学都能快速了解当前模型能力。

平台的核心比喻是“模型超市”：

- 模型像商品一样被平铺展示。
- 每个模型都有封面、能力、合规状态、基础参数、优劣势和评测证据。
- 业务同学可以根据项目需求获得模型推荐。
- 评测同学可以沉淀题库、结果和考点。

当前版本重点是展示和沟通，不是完整工程化系统。它已经具备比较完整的信息架构和交互样式，可作为后续研发接入真实 API、数据库、权限和飞书文档解析的基础。

## 2. 当前包含的页面

### 2.1 模型超市

入口：左侧导航 `🛒 模型超市`

用途：

- 展示当前可用或待评估的图像/视频模型。
- 支持按照图像模型、视频模型分组浏览。
- 每个模型卡片点击后进入模型详情页。

当前展示字段：

- 模型名称。
- 模型类型：Checkpoint、LoRA、Video Base 等。
- 合规状态：业务可用、内部可用、待合规。
- 支持任务：文生图、图生图、编辑、文生视频、图生视频等。
- 媒体类型：图像或视频。
- 性能档位和成本档位。
- 能力标签。
- 适合场景。
- 使用限制。
- 3:4 封面图或视频预览。

当前模型清单：

图像生成模型：

- Seedream 4.5：业务可用。
- Seedream 4.0：业务可用。
- Nano Banana V2：内部可用，不可直接接入业务上线。
- Nano Banana V1：内部可用，不可直接接入业务上线。
- Qwen Edit：业务可用。
- Seed Edit：业务可用。
- Seedream 5.0：待合规，未通过合规前不可业务使用。

视频生成模型：

- Seedance 2.0：业务可用。
- Seedance 2.0 Fast：业务可用。
- Seedance 2.0 Lite：待合规，待测批。
- Seedance 1.0 Mini：业务可用。

合规状态解释：

- 业务可用：已通过内部合规，可接入业务线上流程。
- 内部可用：仅可用于内部工具或探索，不可直接用于业务上线。
- 待合规：确认需要搬运或接入，但尚未完成合规流程，过审前不可业务使用。

### 2.2 模型详情页

入口：

- 点击模型超市中的任意模型卡片。
- 点击需求匹配里的推荐模型卡片。

用途：

- 查看单个模型的介绍。
- 明确模型适合和不适合的场景。
- 绑定相关评测证据。

当前页面结构：

- 左侧为模型封面图。
- 右上角为状态胶囊和返回按钮。
- 模型名称下方展示基础参数，不使用胶囊，避免页面过度拥挤。
- 基础参数包含入参形式、输出尺寸/分辨率、模型类型、合规状态。
- “模型信息”和“能力标签”以轻量布局展示。
- “优势场景”和“使用注意”作为两个独立信息板块展示。
- 下方展示相关评测证据，可跳转到对应题库。

设计注意：

- “使用注意”前的圆点使用红色系，表示风险或限制。
- 业务可用状态胶囊和返回按钮高度一致。
- 当前返回按钮已经放在模型封面卡片右上角区域。

### 2.3 评测题库

入口：左侧导航 `🧪 评测题库`

用途：

- 展示已经沉淀或准备沉淀的评测题库。
- 每张卡片是一个题库入口，而不是单条测试结果。
- 点击题库卡片后进入题库详情页。

当前题库卡片信息：

- 题库标题。
- 更新时间。
- 题库类型：Image、Video、Prompt。
- 业务方向。
- 任务类型。
- 媒体类型。
- 题目总数。
- 预览封面。
- 核心考点。

当前题库包括：

- 海外多元人像：真实导入的文生图 600 道 prompt 题。
- 多语种文字生成：模拟 5 道题。
- 海外本地化丨文化&宗教&节日：模拟 5 道题。
- 2026 Trending 妆造：模拟 5 道题。
- 品牌素材二创与编辑：当前 1 道编辑题。
- 电商商品海报可用性：当前 2 道文生图题。
- 短视频广告镜头生成：当前 2 道文生视频题。

搜索与筛选：

- 支持关键词搜索。
- 支持全部 / 图像 / 视频筛选。
- 筛选器已经改为轻量分段控件，不使用原生下拉样式。

待整理题库：

- 一键试跑中加入的好题会进入待整理题库。
- 待整理题库数量为 0 时不展示卡片。
- 这部分用于承接临时 prompt，后续再人工整理进正式题库。

### 2.4 题库详情页

入口：点击评测题库卡片。

用途：

- 查看某个题库内的题目列表。
- 支持搜索题目。
- 支持选择题目后打开横向对比。
- 支持把 prompt 带到一键试跑。

当前设计：

- 顶部展示题库标题、业务方向、任务类型、媒体类型、题目数量和考点。
- 左侧为题目列表。
- 右侧为选中题目的 prompt、期望能力、考点、输入素材等信息。
- 按钮包括“返回”和“打开横向对比”。

注意：

- “返回”按钮已经缩短为简单文案，放在“打开横向对比”旁边。
- 600 道人像题库当前只展示前 80 条，避免页面过长影响性能。

### 2.5 横向对比

入口：左侧导航 `⚖️ 横向对比`，或从题库详情页打开。

用途：

- 对同一题库内多条 prompt 的多个模型结果进行横向对比。
- 类似图栈逻辑：左侧是 prompt/input，右侧是各模型结果。

当前页面逻辑：

- 顶部可以直接切换题库，不需要返回题库列表。
- 支持切换具体题目组。
- 支持打开“考点高亮”开关。
- Prompt 区不展示额外标题，避免未来新增题时需要 LLM 自动提炼标题。
- Image / Video 类型不重复展示在 prompt 卡片内，因为题库层面已经标注。
- 模型结果列展示输出图/视频、评分、结论、备注。

考点高亮：

- 打开后，prompt 内与考点相关的关键词会用不同颜色标注。
- 当前是前端模拟逻辑，后续可以由模型解析或规则引擎生成。

### 2.6 需求匹配

入口：左侧导航 `✨ 需求匹配`

用途：

- 面向业务方、设计师和产品同学。
- 用户可以输入项目需求，也可以粘贴 PRD / brief / 飞书文档链接。
- 系统解析需求需要关注的考点，并给出模型选型建议。

当前页面结构：

- 需求文本输入框。
- 飞书文档 / 网页链接输入框。
- 需求解析区：媒体类型、任务类型、重点考点。
- 选型建议区：推荐模型、匹配分、推荐理由、证据题库、使用注意。

当前推荐逻辑：

- 前端规则模拟，不是真实 AI 推荐。
- 根据需求文本关键词推断媒体类型、任务类型、考点。
- 根据模型标签、历史结果、合规状态生成候选模型。
- 每个推荐模型必须展示：
  - 为什么推荐。
  - 参考了哪些题库证据。
  - 使用注意。

交互：

- 右上角统计胶囊可跳转到评测题库。
- 证据胶囊可点击跳转到对应评测题库。
- 推荐模型卡片可点击进入模型详情页。

### 2.7 一键试跑

入口：左侧导航 `🚀 一键试跑`

用途：

- 用户输入自己的 prompt。
- 多选模型后同时试跑。
- 临时查看多模型输出。
- 手动给结果打星。
- 将有价值的 prompt 加入待整理题库。

当前流程：

1. 输入 prompt。
2. 设置全局参数。
3. 多选模型，支持全选和清空。
4. 点击开始试跑。
5. 查看横向试跑结果。
6. 用户用星星手动评分，满分 5 分。
7. 可点击“入库为考题”，进入待整理题库。

全局参数：

图像：

- 尺寸比例：1:1、3:4、4:3、9:16、16:9。
- 清晰度：1K、2K、4K。
- 默认：3:4、2K。

视频：

- 尺寸比例：1:1、3:4、4:3、9:16、16:9。
- 分辨率：480P、720P、1080P。
- 默认：9:16、720P。

设计决策：

- 一键试跑不直接选择已有考题，因为考题应该是跑过并沉淀后的正式资产。
- 临时结果不默认显示系统评分，由使用者自己打分。
- 当无结果时，空状态文本居中显示且不换行。

### 2.8 接入指引

入口：左侧导航 `🧭 接入指引`

用途：

- 原“结果标注”页面已调整为接入指引。
- 原因是正式题库中的内容默认已经过人工确认，不应再把“重新标注”作为主入口。
- 这个页面更适合业务同学和设计师理解模型接入与使用边界。

当前内容：

- 业务接入流程。
- 模型状态解释。
- 使用边界和风险。
- 研发/算法对接建议。

## 3. 视觉与品牌设计

网站名称：Model Gem

核心视觉：

- 主色为荧光绿。
- Logo 使用绿色钻石 emoji 视觉。
- 白底、轻量、带一点 editorial / creative tool 气质。

- 使用混排字体设计。
- 左侧词使用花体/衬线感字体，当前字体为 `Griffiths.otf`。
- 右侧重点词使用荧光绿色，并带 1px 右下文字阴影。
- 当前示例：
  - Portrait DEI
  - Global TYPE
  - Local RITE
  - Trend LOOK
  - Brand REMIX

字体文件：

- `public/fonts/Griffiths.otf`

## 9. 后续研发建议

优先级 P0：

- 建立真实模型、题库、题目、结果的数据表。
- 接入真实模型 API 调用。
- 所有试跑结果持久化，不只展示临时结果。
- 引入用户权限：模型管理、业务查看。
- 支持飞书文档链接解析。

优先级 P1：

- 题库批量导入。
- 结果批量导出。
- 视频稳定播放和封面生成。
- 自动初评与人工校正链路。
- 需求匹配接入语义检索。

优先级 P2：

- 推荐解释可视化。
- 多团队协作备注。
- 模型版本变更记录。
- 合规流程状态同步。
- 接入 Byteval 或内部评测平台结果。

## 10. 已知限制

当前 Demo 没有后端，所以：

- 刷新页面后临时试跑结果不会持久保存。
- 一键试跑是模拟结果，不会真实调用模型。
- 需求匹配是前端规则模拟，不是真实 LLM 解析。
- 飞书链接输入框目前只是展示入口，没有读取文档内容。
- 权限切换是前端模拟，没有真实登录。
- 图片和视频素材均为本地占位或参考素材。

## 13. 给研发的建议

技术上建议下一步拆分：

- `data/`：模型、题库、结果 mock 数据。
- `components/`：卡片、页面头、胶囊、状态、评分、预览组件。
- `pages/`：每个导航页面独立文件。
- `services/`：模型 API、推荐、飞书解析、导入导出。
- `types/`：统一类型定义。

当前 `App.tsx` 承载了大部分逻辑，适合 Demo 快速迭代；如果进入研发阶段，建议尽快拆分，避免继续堆大文件。
