> **一句话说明**  
> Model Gem 是一个面向 TikTok 内部的模型效果评测与选型 Demo。它把模型超市、评测题库、横向对比、需求匹配、一键试跑和接入指引串成一个可演示的网站，用于帮助设计、产品、算法、研发和业务同学快速判断“哪个模型适合当前需求，为什么，证据是什么”。

| --- | --- | --- |

### 当前版本不是完整后台系统

> **重要边界**  
> 当前项目是前端 Demo。它适合做方案沟通、设计评审、跨团队对齐和研发需求澄清；还没有真实后端、真实模型 API、真实登录鉴权和真实飞书文档解析。

当前暂不包含：

- 真实模型 API 调用。
- 真实数据库和持久化。
- 真实用户登录和权限控制。
- 真实文件上传。
- 真实飞书文档内容解析。
- 真实合规系统状态同步。

## 1. 产品目标

Model Gem 的目标不是再做一个“模型列表”，而是把模型能力和评测证据绑定起来。

核心目标可以拆成三层：

1. **看得懂**：业务、设计、产品可以快速知道每个模型是什么、能做什么、能不能上线用。
2. **比得清**：同一组 prompt / input 下，多个模型的输出效果、评分、结论和问题能横向比较。
3. **选得准**：业务输入需求后，系统根据考点、历史题库和模型状态推荐候选模型，并说明推荐理由和使用注意。

## 4. 页面总览

### 4.1 模型超市

**定位**：网站第一入口，把模型像“商品货架”一样展示。

页面展示：

- 图像模型和视频模型分区。
- 每个模型有 3:4 封面图或视频封面。
- 模型卡片点击后进入模型详情页。
- 卡片展示模型名称、类型、状态、任务、能力标签、成本和性能信息。

当前模型：

| 类型 | 模型 | 状态 | 说明 |
| --- | --- | --- | --- |
| 图像 | Seedream 4.5 | 业务可用 | 可作为业务主流程参考 |
| 图像 | Seedream 4.0 | 业务可用 | 稳定成熟，适合批量任务 |
| 图像 | Nano Banana V2 | 内部可用 | 只能内部工具使用，不可直接业务上线 |
| 图像 | Nano Banana V1 | 内部可用 | 只能内部工具使用，不可直接业务上线 |
| 图像 | Qwen Edit | 业务可用 | 偏编辑和图生图任务 |
| 图像 | Seed Edit | 业务可用 | 偏素材编辑和主体保持 |
| 图像 | Seedream 5.0 | 待合规 | 未通过合规前不可业务使用 |
| 视频 | Seedance 2.0 | 业务可用 | 视频主流程参考 |
| 视频 | Seedance 2.0 Fast | 业务可用 | 偏快速生成 |
| 视频 | Seedance 2.0 Lite | 待合规 | 待测批 |
| 视频 | Seedance 1.0 Mini | 业务可用 | 轻量版本 |

### 4.2 模型详情页

**定位**：单个模型的说明页，用于回答“这个模型适合我吗”。

页面重点：

- 模型封面。
- 状态与返回按钮。
- 基础参数：入参形式、输出尺寸/分辨率、模型类型、合规状态。
- 模型信息。
- 能力标签。
- 优势场景。
- 使用注意。
- 相关评测证据。

> **设计原则**  
> 详情页已经减少胶囊数量。模型名称下方的基础参数采用轻量文字结构，避免页面信息全部变成标签。

### 4.3 评测题库

**定位**：所有评测资产的入口页。这里展示的是“题库”，不是单条测试结果。

当前题库：

| 题库 | 类型 | 数量 | 状态 |
| --- | --- | --- | --- |
| 海外多元人像 | 文生图 | 600 道 | 真实导入 |
| 多语种文字生成 | 文生图 | 5 道 | 模拟题 |
| 海外本地化丨文化&宗教&节日 | 文生图 | 5 道 | 模拟题 |
| 2026 Trending 妆造 | 文生图 | 5 道 | 模拟题 |
| 品牌素材二创与编辑 | 编辑 | 1 道 | 模拟题 |
| 电商商品海报可用性 | 文生图 | 2 道 | 模拟题 |
| 短视频广告镜头生成 | 文生视频 | 2 道 | 模拟题 |

卡片逻辑：

- 每张卡片是题库入口。
- Prompt 类型题库使用文字封面。
- Image / Video 类型题库使用图片或视频预览。
- 待整理题库只在存在待整理题目时展示。

### 4.4 题库详情页

**定位**：查看某个题库里的题目列表和单条 prompt 信息。

页面包含：

- 题库基础信息。
- 题目搜索。
- 题目列表。
- 选中题目的 prompt。
- 期望能力。
- 关联考点。
- 操作按钮：返回、打开横向对比、带入一键试跑。

> **注意**  
> “海外多元人像”虽然有 600 道题，但详情页默认只展示前 80 条，避免页面过长影响性能。

### 4.5 横向对比

**定位**：核心评测看板，用于看清“同题不同模型”的效果差异。

当前逻辑：

- 左侧是 prompt / input 内容。
- 右侧横向排列多个模型的输出结果。
- 支持在页面内切换题库。
- 支持考点高亮开关。
- 不为每条 prompt 自动生成小标题，避免未来运行成本增加。

适合回答：

- 哪个模型在同一条 prompt 下表现更好？
- 哪些模型容易出现文字错误、主体崩坏、风格漂移？
- 人工评分和最终结论是什么？
- 某个题库是否能作为业务参考证据？

### 4.6 需求匹配

**定位**：面向业务和设计师的“选型建议入口”。

用户可以输入：

- 一段业务需求。
- 一段 brief。
- PRD 摘要。
- 飞书文档链接。
- 任意网页链接。

系统输出：

- 媒体类型。
- 任务类型。
- 重点考点。
- 推荐模型。
- 推荐理由。
- 证据题库。
- 使用注意。

> **当前实现方式**  
> 这是前端规则模拟。真实版本建议接入飞书文档解析、语义检索和模型推荐服务。

### 4.7 一键试跑

**定位**：临时验证新 prompt 的实验台。

流程：

1. 输入 prompt。
2. 设置全局参数。
3. 多选模型。
4. 开始试跑。
5. 横向查看结果。
6. 用户手动打 1-5 星。
7. 好题可加入待整理题库。

默认参数：

| 媒体 | 参数 | 默认值 |
| --- | --- | --- |
| 图像 | 尺寸比例 | 3:4 |
| 图像 | 清晰度 | 2K |
| 视频 | 尺寸比例 | 9:16 |
| 视频 | 分辨率 | 720P |

> **产品决策**  
> 一键试跑不让用户直接选择已有考题，因为正式考题应当是已经跑过、整理过、能够复用的资产。

### 4.8 接入指引

**定位**：替代原来的“结果标注”页面。

调整原因：

- 正式进入评测题库的内容，默认已经完成过人工确认。
- 重新标注不应该成为主入口。
- 对业务和设计师来说，更重要的是理解模型是否能接入、如何接入、有什么风险。

页面用于说明：

- 业务接入流程。
- 模型状态边界。
- API / 参数 / 风险 / 复查路径。
- 业务可用、内部可用、待合规的区别。

## 11. 已知限制

> **当前限制汇总**  
> 当前版本适合演示和继续设计，不建议直接作为生产系统使用。

- 刷新页面后临时试跑结果不会持久保存。
- 一键试跑是模拟结果，不会真实调用模型。
- 需求匹配是前端规则模拟，不是真实 LLM 解析。
- 飞书链接输入框目前只是展示入口，没有读取文档内容。
- 权限切换是前端模拟，没有真实登录。
- 图片和视频素材均为本地占位或参考素材。

## 12. 给下一位设计师

## 13. 给研发

当前 `App.tsx` 承载了大部分 Demo 逻辑，适合快速迭代，但不适合继续长期堆功能。

建议拆分：

```text
src/
  data/          # mock 数据
  types/         # 类型定义
  components/    # 卡片、状态、评分、预览组件
  pages/         # 每个导航页面
  services/      # API、推荐、飞书解析、导入导出
  utils/         # 格式化、筛选、映射逻辑
```

建议优先抽离：

- 模型数据。
- 题库数据。
- 测试结果数据。
- 推荐逻辑。
- 页面头部和统计组件。
- 卡片组件。
- 星级评分组件。
- Prompt 高亮组件。
